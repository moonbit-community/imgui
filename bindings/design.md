# Binding Design

This document is the source of truth for the MoonBit Dear ImGui binding ABI
rules and package boundaries.

## Package Layout

The repository must expose exactly two MoonBit packages:

- `moonbit-community/imgui/bindings`
- `moonbit-community/imgui`

No other MoonBit packages are allowed. In particular, there must be no `raw`,
`generated`, `advanced`, backend, or example package.

`moonbit-community/imgui/bindings` is the pure generated binding package for the
C API emitted by `dear_bindings`. It must not contain hand-written C, C++, or
MoonBit helper code. It must not use custom C/C++ stubs to repair ABI cases.

`moonbit-community/imgui` is the user-facing wrapper package built on top of
`moonbit-community/imgui/bindings`.

## Binding Source

The binding target is the C API generated by `dear_bindings` for Dear ImGui.
The binding package must bind that generated C API directly.

Generated Dear Bindings output such as `dcimgui.h`, `dcimgui.cpp`, and metadata
JSON may be checked in. Upstream Dear ImGui source may be used as upstream
source. Hand-written bridge C/C++ code is not allowed.

## MoonBit Native ABI

MoonBit native ABI mapping used by this project:

| MoonBit type | Native ABI |
| --- | --- |
| `Bool` | `int32_t` |
| `Int` | `int32_t` |
| `UInt` | `uint32_t` |
| `Int64` | `int64_t` |
| `UInt64` | `uint64_t` |
| `Float` | `float` |
| `Double` | `double` |
| constant enum | `int32_t` |
| abstract type, `type T` | pointer to a valid MoonBit object |
| external type, `#external type T` | `void*` |
| `FixedArray[Byte]` / `Bytes` | `uint8_t*` with trailing `\0` |
| `FixedArray[T]` | `T*` |
| `FuncRef[T]` | function pointer |
| `Ref[T]` | `T*` |
| `Option[Ref[T]]` | nullable `T*`, where `None` is `NULL` |

A foreign function returning `Unit` corresponds to a C function returning
`void`. If the return type inside `FuncRef[T]` is `Unit`, the function pointer
returns `void`.

## Type Mapping Rules

Primitive C values map to the corresponding MoonBit primitive from the ABI
table.

C constant enums map to `Int`.

Non-null C pointers to primitive values map to `Ref[T]`.

Nullable C pointers to primitive values map to `Option[Ref[T]]`.

MoonBit `String` must not be used in `bindings` for C `char*`. MoonBit `String`
is UTF-16. C UTF-8 strings must use `Bytes` for non-null `char*` / `const char*`.
Generated bindings must not expose custom `CString` or generated pointer wrapper
types for user-visible string ABI.

`#external type T` represents a C pointer handle directly. For a C pointer whose
pointee is represented by a generated `#external type`, the binding type is `T`,
not `Ref[T]`, `Option[T]`, or `Option[Ref[T]]`.

Every generated `#external type T` must also expose:

```moonbit
T::null() -> T
T::is_null(T) -> Bool
```

Nullable external handles are represented by `T::null()` and tested with
`T::is_null(value)`. Do not use `Option[T]` for external handles.

If MoonBit does not provide a built-in null constructor for external types, the
generator may emit a small generated C support file that returns `NULL` for
`T::null()`. This file is generated ABI support, not hand-written bridge code.
`T::is_null` should bind to the MoonBit runtime `moonbit_is_null(void*)`
function when available.

C function pointer typedefs map to `FuncRef[...]` when they are ABI-direct.
They must not be represented by opaque pointer wrapper types.

## Generated API Rules

The generated `bindings` API must not use MoonBit optional arguments or default
arguments. C defaults from `dear_bindings` metadata must become explicit
MoonBit parameters.

The generated `bindings` API must not expose names such as
`@generated.FloatPtr`, `@generated.CString`, or other custom helper pointer
types.

The generated `bindings` API should preserve direct C ABI names in idiomatic
MoonBit snake case, for example `ImGui_Begin` becomes `im_gui_begin`.

The user-facing `imgui` package should remove redundant `imgui` prefixes from
its public functions because the package name already provides that namespace.

FFI pointer parameters must use explicit `#borrow` or `#owned` annotations.
Borrowing is the default policy for direct bindings unless ownership transfer is
documented by Dear ImGui.

## Unsupported ABI Shapes

If a C API shape cannot be expressed directly with the MoonBit ABI above, the
generator must report it in the coverage report instead of adding a custom C/C++
bridge. Examples include C varargs and C++ structs returned by value, unless
MoonBit gains a direct ABI representation for that shape.

The coverage report must make unsupported symbols visible and explain the
reason.

## Wrapper Rules

The top-level `imgui` package may provide hand-written MoonBit wrappers over
`imgui/bindings`, but it must not leak generated pointer helper types or string
carrier types into the user-facing API.

The wrapper should keep API names concise and idiomatic for MoonBit users.
